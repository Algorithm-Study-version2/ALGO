## 접근
나이브한 아이디어로는, 매 i번째 쿼리마다
크기 2의 윈도우를 0...size-2까지 순회해
색상쌍의 갯수를 구한다.
그러나

- ```1 <= n <= 10^5```
- ```1 <= queries.length <= 10^5```

에서 ```n*queries.length <= 10^10``` 이므로 불가.
적어도 ```N * ㅣog N``` 이하의 방법을 찾아야 함

매 쿼리마다 갱신하는 인덱스에 대해서

- 해당 인덱스의 전후에 대해
 같은 색이라면 색상쌍의 갯수가 줄으므로 
 전-후 각 count-1
- 갱신 후 인덱스의 전후에 대해
 같은 색이라면 색상쌍의 갯수가 느므로
 전-후 각 count+1

count는 이전 쿼리까지의 색상쌍 갯수를 누적하고 있으므로
상기 방법은 유효.
