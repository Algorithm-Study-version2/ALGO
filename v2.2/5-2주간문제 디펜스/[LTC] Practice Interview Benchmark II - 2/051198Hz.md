## 풀이

`"트리 + prefix xor + parity bitmask"`

재배열 펠린드롬의 조건:
`문자열 내 홀수 개 문자가 최대 1개`

각 문자의 짝홀수를 0,1로 하여 마스크로 표현
```
등장 횟수:
a: 2
b: 1
c: 1

parity:
a -> 0
b -> 1
c -> 1

bitmask:
110
```

이제 루트부터 각 노드까지의 마스크를 정의:
```
mask[node]
root -> node 경로에서
각 문자 등장 parity

예를들어,

root -> x

경로가 a,b,a,c 라 하면

b,c는 홀수
mask = 1100
```

XOR은 1 XOR 1 == 0 , 1 XOR 0 == 1 이므로,
문자 갯수를 더하면서도 홀수개를 표시할 수 있다.

따라서, u -> v로 이동하면서 mask[u] XOR mask[v] 한다.

![ex1](https://assets.leetcode.com/uploads/2023/07/15/treedrawio-8drawio.png)

위 그림에서 3 -> 4 일 때에도 mask[3] XOR mask[4]가 성립하는 이유는
XOR시 짝수는 모두 0처리 되고, 공통인 부분은 짝수이므로 0처리 된다.
따라서 3 -> 4 일때의 문자열만 검사하게 된다.

이후 DFS하며 루트로부터 만들수 있는 모든 마스크를 구한다.
